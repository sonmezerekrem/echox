## v1.3.3

*Released: February 2026*

### Fixes

- **Dev server live reload** — When running `echox dev` from your docs folder, edits to markdown and other files now update the UI. Content, APIs, and assets live outside the Astro project root; the dev server now watches `content/`, `apis/`, and `assets/` so changes trigger a full reload.

---

## v1.3.1

*Released: March 1, 2026*

### Fixes

- **Base path support** — All internal links, redirects, and assets now respect `ECHOX_BASE` (e.g. `/echox/` on GitHub Pages). Fixes 404s when the site is served from a subpath.
- **Deploy cleanup** — GitHub Actions workflow removes old deployment records after each deploy so the Deployments tab stays clean.

### Changes

- Dynamic base path applied in Sidebar, TabNav, PrevNextNav, HeaderActions, Footer, 404 page, and TaskCard links.
- `echox build` with `ECHOX_BASE` set produces a site that works correctly when deployed to GitHub Pages or any subpath.

## v1.3.0

*Released: March 1, 2026*

### Features

- **Version command** — `echox -v`, `echox --version`, and `echox version` print the installed version from package.json.
- **GitHub Pages deploy** — New workflow deploys the example docs to GitHub Pages on push to `main`. Docs live at `https://<user>.github.io/<repo>/`.
- **Init improvements** — `echox init` now creates:
  - `.gitignore` with common ignores (e.g. `.env`, `node_modules`, `dist`).
  - `Dockerfile` (build + serve with nginx) and `.dockerignore`.
  - `git init` if the project is not already a git repository.
  - Sample `tasks.yml` and OpenAPI `apis/sample_api.json` (unchanged).

### Fixes

- **Build from user directory** — Astro is now spawned with `cwd` set to the Echox package root so dependencies (e.g. Piccolore) resolve correctly when running from a user docs directory.
